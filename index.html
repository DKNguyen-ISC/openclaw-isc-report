3 Gaps That Need to Close 🔒

These are not permanent dealbreakers — they are gaps that the industry is actively working to close. Each one maps directly to ISC's current infrastructure, data sensitivity, and operational scale. When these gaps are resolved, our assessment will change — and DK will report it immediately.

🔒
Gap #1 — Security Transparency

No Enterprise Security Audit Exists Yet

The Situation

To automate tasks, OpenClaw needs deep access to your computer — files, email, databases. ISC's M1-M4 SCM system holds pricing logic, customer data, and supplier relationships. Any tool with this level of access needs a proven security track record.

What's Currently Missing

  • ❌ No SOC 2 or ISO 27001 certification
  • ❌ No third-party security audit published
  • ❌ Open-source code is transparent but unaudited
  • ❌ No formal data handling guarantees
What would change our assessment: An enterprise security audit (SOC 2 / ISO 27001), or a major corporation publicly deploying it with documented security review. DK is monitoring this.
🖥️
Gap #2 — Infrastructure Requirements

Dedicated Hardware Needed for Safe Operation

The Situation

The safest way to run any AI agent is on an isolated computer — not connected to ISC's main network. This prevents accidental data exposure. Installing it on daily work computers creates a direct path to all company data.

What ISC Would Need

  • 🖥️ 1 dedicated isolated server or workstation
  • 🔐 Network isolation (firewall, separate VLAN)
  • 🛠️ IT maintenance for the isolated environment
  • 💵 Estimated: $2,000–$5,000 USD upfront
What would change our assessment: Cloud-hosted AI agent services (like Gemini or Claude offering managed agent environments) would eliminate the hardware requirement entirely. This is already happening.
💸
Gap #3 — Cost Predictability

No Clear Budget Control Mechanism

Option A — Pay-per-use AI

  • 📊 Every task calls an AI API — billed per token
  • ⏰ 24/7 operation = thousands of calls/day
  • 💰 Monthly estimate: $500–$3,000+ USD
  • ❓ Cost depends on task complexity — unpredictable

Option B — Free Local AI

  • ✅ No API cost — but GPU server needed: $3,000–$8,000
  • ⚠️ Current free models lack accuracy for SCM data
  • 📉 Risk of logic errors in critical calculations
  • 📈 Improving rapidly — this gap is actively shrinking
What would change our assessment: Fixed-price enterprise subscriptions from AI vendors, or local models reaching sufficient accuracy for SCM calculations. Both are expected within 12–18 months.

The Logic — ISC Shortage Engine

BigQuery SQL · shortage_engine.sql 
-- ISC Shortage Engine (Google BigQuery)
SELECT
  Part_ID,
  Part_Name,
  Demand_This_Week      AS Demand,
  Current_Stock         AS Stock,
  On_The_Way_PO         AS PO_Supply,
  (Demand - Stock - On_The_Way_PO) AS SHORTAGE
FROM SCM_Database
WHERE (Demand - Stock - On_The_Way_PO) > 0
ORDER BY SHORTAGE DESC;
